# Project Delivery Checklist

*Use before every project handoff*

---

## Pre-Delivery (Day Before)

- [ ] Run final QA on all features
- [ ] Test on multiple browsers/devices
- [ ] Verify all links work
- [ ] Check responsive design
- [ ] Test forms and validations
- [ ] Verify database/data integrity
- [ ] Check error logging working

---

## Files & Code

- [ ] Source code committed to repo (GitHub/GitLab)
- [ ] Branch naming convention followed
- [ ] README with setup instructions
- [ ] Environment variables documented
- [ ] All secrets removed from code
- [ ] No hardcoded credentials

---

## Hosting & Infrastructure

- [ ] Domain pointing to correct server
- [ ] SSL certificate installed
- [ ] Hosting account transferred/accessible
- [ ] Database backed up
- [ ] CI/CD pipeline configured
- [ ] Backup strategy in place

---

## Third-Party Services

- [ ] API keys configured for production
- [ ] Payment gateway in live mode
- [ ] Email/SMS services working
- [ ] Analytics tracking verified
- [ ] Webhooks tested

---

## Documentation

- [ ] User guide / documentation
- [ ] Admin guide (if applicable)
- [ ] API documentation (if applicable)
- [ ] Deployment steps
- [ ] Tech stack summary

---

## Client Handoff

- [ ] Access credentials shared securely
- [ ] Demo call scheduled
- [ ] Training call completed
- [ ] Support channels communicated
- [ ] Warranty period clarified

---

## Admin/Internal

- [ ] Invoice sent for final payment
- [ ] Project marked complete in tracker
- [ ] Internal retrospective done
- [ ] Portfolio case study started
- [ ] Testimonials requested

---

*Tip: Use this as a Notion template or GitHub issue checklist for each project.*
